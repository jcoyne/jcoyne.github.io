Notes for forms, presenters, indexers

Indexers:

  • New part of ActiveFedora 9 (Jan 2015)
  • Replaces overriding to_solr
  • Put a Fedora model into Solr
  • Keeps the concern of "how to index" out of the model.
  • Supports testing

Presenter

  • Interface between Controller - View
  • Represents the model in a specific context (e.g. the show page)
  • Can have multiple presenters for each object class
  • Could wrap a Fedora model, but in Hydra we get better perfomance if we wrap a SolrDocument
  • Could be use to autogenerate part of the view page
  • Can replace Helpers

Index + Presenter = BFF

  • Indexer - Map to solr
  • Presenter - Map from solr
  • WARNING: lossy!

Form

  • Represent the edit page.
  • Interface between Controller - View
  • Input for the update action
  • Could be use to autogenerate the form
